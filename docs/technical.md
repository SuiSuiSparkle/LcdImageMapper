# ImageTool 技术文档

## 1. 项目目标

ImageTool 是一个图像取模工具，支持单张图片和文件夹批处理，将 PNG、JPG/JPEG、BMP 图像转换为适合嵌入式显示的 C 数组。

## 2. 核心能力

- 图像加载后自动转为灰度图。
- 支持阈值二值化，阈值范围为 0 到 255。
- 支持 Horizontal / Vertical 两种扫描方向。
- 支持 MSB first / LSB first 两种字节位顺序。
- 支持自动补齐到 8 的倍数。
- 支持单图输出、批处理结果复制、批量 TXT 导出。

## 3. 性能优化

### 3.1 向量化二值化

当前版本使用 NumPy 的矩阵比较完成二值化，而不是逐像素 Python 循环：

$$
binary = gray >= threshold
$$

这可以显著减少大图换图时的 UI 卡顿。

### 3.2 向量化打包

位打包使用 `np.packbits` 完成。相比 Python 手写位移循环，打包过程更快，尤其在批处理时收益明显。

### 3.3 后台批处理

文件夹批处理通过 `QThread` 放到后台执行，主线程只负责界面刷新和结果展示。这样即使图片数量较多，也不会长时间阻塞窗口响应。

## 4. 独显优先

程序启动时会设置 Qt 的桌面 OpenGL 偏好，用于尽量让窗口渲染走硬件加速路径：

- `QT_OPENGL=desktop`
- `AA_UseDesktopOpenGL`

说明：是否真正使用独立显卡，最终仍受 Windows 图形设置、显卡驱动和 Qt/OpenGL 环境影响。如果需要强制某个显卡，建议在 Windows 的图形设置里把程序指定为高性能模式。

## 5. 批处理规则

### 5.1 文件夹校验

文件夹批处理要求：

- 文件夹内只能包含支持的图片文件。
- 文件夹内的图片必须是同一种扩展名。
- 如果发现混合格式、子文件夹或非图片文件，程序会直接提示错误。

### 5.2 双向链表存储

批处理结果会按序存放到双向链表中，节点包含：

- 序号
- 文件路径
- 原始尺寸
- 补齐尺寸
- 结果文本
- 字节数量

这样可以快速按序号定位某一张图片的结果。

### 5.3 TXT 导出

批处理导出 TXT 时，用户可以自定义分隔符。所有图片结果会按链表顺序拼接后输出到一个文本文件里。

## 6. 输出格式

生成内容包含：

- `#define IMG_WIDTH...`
- `#define IMG_HEIGHT...`
- `#define IMG_BYTES...`
- `const uint8_t bitmap...[] = { ... };`

数组内容按每 16 字节一行输出，并在行首带偏移注释，便于复制和调试。

## 7. 代码结构

- `imagetool.py`：主程序，包含 UI、批处理线程、取模算法、复制和导出逻辑。
- `docs/usage.md`：操作指南。

## 8. 扩展建议

- 如果后续要支持反色、旋转或裁剪，建议把图像预处理再拆成独立函数。
- 如果后续要支持更大的图片集，可把批处理结果改成懒加载，减少内存占用。
