# ImageTool 操作指南

## 1. 启动程序

在项目目录下运行：

```bash
py -3 imagetool.py
```

如果系统已经把 Python 加入 PATH，也可以直接运行 `python imagetool.py`。

## 2. 打开单张图片

1. 点击 `Open Image`。
2. 选择 PNG、JPG/JPEG 或 BMP 文件。
3. 左侧显示原图，右侧显示二值点阵预览。

## 3. 打开文件夹批处理

1. 点击 `Open Folder`。
2. 选择一个只包含同类型图片的文件夹。
3. 程序会自动检查文件夹是否混入其他格式或非图片文件。
4. 如果校验通过，程序会在后台批量生成结果，避免界面卡死。

## 4. 调整运算参数

- `Threshold`：调节灰度二值化阈值。
- `Horizontal / Vertical`：切换扫描方向。
- `MSB first / LSB first`：切换字节内位顺序。

在批处理模式下修改这些参数，会重新触发后台批处理，以保证所有结果一致。

## 5. 选择批处理结果

当文件夹加载完成后，可以在 `Sequence` 中选择某一张图片的序号：

- 序号对应文件夹内的排序结果。
- 选择不同序号后，输出框会显示对应图片的取模结果。
- `Copy` 会复制当前序号对应的结果。

## 6. 复制结果

- 单图模式：复制当前图片的 C 数组代码。
- 批处理模式：复制当前选中序号的结果。

## 7. 保存头文件

点击 `Save as .h` 后，会把当前显示的结果保存为头文件。

## 8. 导出 TXT 文件

批处理模式下可以填写 `Delimiter`，用于分隔不同图片对应的结果文本。

1. 在 `Delimiter` 中输入你想要的分隔符。
2. 点击 `Export TXT`。
3. 选择保存位置后，程序会把全部结果按顺序写入一个 `.txt` 文件。

## 9. 性能说明

- 大图换图不再使用逐像素 Python 循环，因此响应更快。
- 批处理在后台线程执行，窗口不会长时间卡死。
- 如果你希望进一步减少 UI 压力，可以先缩小图片尺寸再导入。

## 10. 常见问题

### Q: 为什么文件夹里不能混放不同图片格式？
A: 这是为了保证批处理输出规则一致，也方便一次性生成统一结果。

### Q: 为什么批处理结果会重新生成？
A: 因为阈值、扫描方向和位顺序改变后，所有结果都需要保持一致，所以程序会重新批处理。

### Q: 独立显卡一定会被使用吗？
A: 不一定。程序已经尽量启用桌面 OpenGL，但最终仍受 Windows 图形设置和驱动影响。建议在系统图形设置里把该程序指定为高性能模式。

### Q: 输出文本可以再分组吗？
A: 可以。你可以通过 `Delimiter` 自定义图片结果之间的分隔文本。
